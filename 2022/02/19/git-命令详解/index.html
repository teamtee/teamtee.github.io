git 命令详解

发表于2022-02-19|更新于2022-02-19
|字数总计:2k|阅读时长:7分钟|阅读量:

一、配置

git有system config、global config、local config有3种配置文件，配置文件生效的范围为：系统（所有用户）、全局（当前用户）、本地；生效的配置文件为当前文件夹距离最小的配置文件，local>global>system

1.查看配置

git config [–local|–global|–system] –list

git config –list #显示本地、全局、系统配置
git config –global –list #显示全局配置
git config –list 显示顺序

2.改变配置

git config [–local|–global|–system] section.key value

git config –global user.email “you@example.com” #改变全局用户邮件
git config –global user.name “Your Name” #改变全局邮箱
git config –global merge.tool vimdiff #冲突差异分析工具
git config –global core.editor emacs #默认编辑器

3.增加配置

git config [–local|–global|–system] –add section.key value

git config –add alias.cf config #alias是别名的意思，该条为config建立了一个叫cf的别名

4.删除配置

git config [–local|–global|–system] –unset section.key

5.查看某一配置

git config [–local|–global|–system] –get section.key

二、存储

工作目录（Working Directory）、暂存区(Stage/Index)、资源库(Repository或Git Directory)、远程的git仓库(Remote Directory）

![20210731100735629 ](C:\Users\upshift\Desktop\git\git_picture\20210731100735629 (1).png)

1.git add

可以是文件名或者路径

git add . # . 代表所有文件
git add * # 添加所有文件
git add hello.txt # 提交文件hello.txt
git add documentation/ # 将document文件下的所有文件提交
git add documentation/*.txt # 将document文件下的所有txt文件提交
git add Controller # 将以Controller结尾的文件的所有修改添加到暂存区
git add Hello # 将所有以Hello开头的文件的修改添加到暂存区
git add Hello? # 将以Hello开头后面只有一位的文件的修改提交到暂存区 例

2.git checkout

git checkout –

可以是文件名或者路径,表示从暂存区覆盖工作区，–可以省略

git checkout –

从branch拉取文件，覆盖暂存区和工作区文件，–可以省略

git checkout hello.c #从暂存区（别人也称索引）回复文件hello.c
git checkout – hello.c #效果同上，但是这是为了防止有一个分支名为hello.c
git checkout – ‘*.c’ #’’不必须
git checkout master hello.txt #从master分支覆盖当前工作区和暂存区的hello.txt文件

git checout

切换分支

git checkout master #切换到master分支
git checkout HEAD^^ #切换到当前分支的前2次提交
git checkout –amend #撤销上一次提交，并且重新commit

git checkout -b

git checkout -b teamtee #从当前分支创建并且切换到teamtee分支
git checkout -b newBranch origin/newBranch
#从origin/newBranch分支创建并且切换到本地的newBranch分支

git checkout HEAD~ #切换到A3分支
git checkout HEAD^ #切换到A3分支
git checkout HEAD1 #切换到A3分支
git checkout HEAD~~ #切换到A2分支
git checkout HEAD^^ #切换到A2分支
git checkout HEAD2 #切换到A2分支
git checkout HEAD^1 #切换到B2分支
git checkout HEAD^11 #切换到B1分支
git checkout HEAD^22 #切换到C1分支

3.git commit

git commit -m #提交暂存区修改和中的信息到本地仓库

git commit -am #相当于git add . 加上 git commit -m ，但不同的是这里
的git add .只是会提交所有在暂存区已经有了的文件的修改,
即tracked文件。
git commit [file1] … -m #提交指定文件

4.git rm

git rm |-r

git rm #从工作区和暂存区移除file,
git rm -r #从工作区和暂存区移除文件夹folder
git rm -f #如果文件修改后已经add，必须用强制删除
git rm –cached #删除暂存区
rm #删除工作区

5.git reset

git reset [-hard|-soft]

将当前的HEAD和HEAD所指向的分支移动到新的分支

git reset #清空暂存区，覆盖工作区
git reset -hard #同时覆盖暂存区和工作区
git reset -soft #覆盖暂存区

6.git revert

git revert

产生一次新的commit，但是去除了内这次提交对内容的修改

分支切换查看

git log:从当前branch向上查看commit

git reflog:查看所有提交与分支切换信息

(参考https://blog.csdn.net/ouyang_peng/article/details/84061662）

7.git push

git push <远程主机名> <本地分支名>:<远程分支名>

git push origin master:master #从本地的master分支推送到origin主机的master分支
git push origin master #将本地的master分支推送到origin主机的master分支。如果
master不存在，则会被新建
git push origin #从本地的当前分支推送到origin有追踪关系的分支
git push #只有origin一个主机，或是设置了默认主机，推送到追踪分支
git push -u origin #设置默认主机为origin同时推送
git pus origin :master #从本地推送一个空的分支到到origin主机的master分支
git push origin –delete master #等价于上面一条
git push –all origin #推送所有分支
git push –force origin #如果远程分支比当前分支新，必须用强制指令直接覆盖
git push origin –tags #推送tag

8.git pull

git pull <远程主机名> <远程分支名>:<本地分支名>

git pull用给定的参数运行git fetch，并调用git merge将检索到的分支头合并到当前分支中

9.git fetch

git fetch <远程主机名> <分支名>

git fetch origin master #取回origin 主机的master 分支
git fetch origin #取回全部分支
git fetch git://git.git mster #从指定位置取回master

取回更新后，会返回一个FETCH_HEAD ，指的是某个branch在服务器上的最新状态，我们可以在本地通过它查看刚取回的更新信息：

$ git log -p FETCH_HEAD

10.git branch

git branch #查看本地分支
git branch -r #查看远程分支
git branch -a #查看和本地远程分支
git branch -d #删除本地分支
git branch -d -r #删除远程分支，删除后还需推送到服务器
git branch #创建分支
git branch -m #修改分支名

11.git merge

git merge #和并branch到当前分支
git merge
#合并分支

12.git init

git init #直接在当前文件夹内建立.git文件
git init #在路径内建立.git文件

13.git clone

git clone #url甚至可以为本地

14.git remote

git remote -v #显示所有远程版本库
git remote show #显示远程版本库信息
git remote add #添加远程库
git remote rm name # 删除远程仓库
git remote rename old_name new_name # 修改仓库名

15.git rebase

git rebase #将当前分支从与branch分岔的地方开始移动到branch后
https://www.yiibai.com/git/git_rebase.html

16.git commit

(1)提交

git commit -m

git commit -a -m #相当于git add +commit

（2）修改 –amend

git commit -amend

17.git stash

https://www.yiibai.com/git/git_stash.html

(1)合并分支再提交

三.应用

1.绑定本地仓库和远程仓库

git push –set-upstream origin #将当前分支绑定到origin库，新建branch分支

git checkout -b newbranch origin branch在本地建立并且于远端分支建立联系

2.版本回退

（1）查看历次提交

git log：查看历次提交信息

git reflog：查看所有提交信息，包括已经删除的提交

（2）回退命令

git reset –hard ：不丢失commit

git reset –soft：默认soft，丢失·commit

git revert：反向提交

git reset 是把HEAD向后移动了一下，而git revert是HEAD继续前进，只是新的commit的内容和要revert的内容正好相反，能够抵消要被revert的内容。
